## 摘要
在非结构化环境中，实现具有功能性的灵巧抓取需要语义理解、精确的三维功能定位以及物理可解释的执行之间的紧密融合。模块化分层方法相比端到端的视觉-语言-动作（VLA）方法更具可控性和可解释性，但现有方法仍依赖预定义的可供性标签，缺乏功能性灵巧操作所需的语义—位姿耦合。为此，我们提出了 BLaDA（Bridging Language to Dexterous Actions in 3DGS fields），一个可解释的零样本框架，它将开放词汇的指令作为功能性灵巧操作的感知与控制约束进行落地。BLaDA 通过一个知识引导语言解析（Knowledge-guided Language Parsing, KLP）模块，首先将自然语言解析为一个包含操作约束的结构化六元组，从而建立可解释的推理链。为了实现与位姿一致的空间推理，我们引入了三角功能点定位（Triangular Functional Point Localization, TriLocation）模块，利用三维高斯喷溅（3D Gaussian Splatting）作为连续场景表示，并在三角几何约束下识别功能区域。最后，三维关键点抓取矩阵变换执行（3D Keypoint Grasp Matrix Transformation Execution, KGT3D+）模块将这些语义—几何约束解码为物理可行的腕部位姿和指级命令。在复杂基准上的大量实验表明，BLaDA 在可供性落地精度和跨不同类别与任务的功能操作成功率方面均显著优于现有方法。代码将公开于 https://github.com/PopeyePxx/BLaDA。

## Abstract
In unstructured environments, functional dexterous grasping calls for the tight integration of semantic understanding, precise 3D functional localization, and physically interpretable execution. Modular hierarchical methods are more controllable and interpretable than end-to-end VLA approaches, but existing ones still rely on predefined affordance labels and lack the tight semantic--pose coupling needed for functional dexterous manipulation. To address this, we propose BLaDA (Bridging Language to Dexterous Actions in 3DGS fields), an interpretable zero-shot framework that grounds open-vocabulary instructions as perceptual and control constraints for functional dexterous manipulation. BLaDA establishes an interpretable reasoning chain by first parsing natural language into a structured sextuple of manipulation constraints via a Knowledge-guided Language Parsing (KLP) module. To achieve pose-consistent spatial reasoning, we introduce the Triangular Functional Point Localization (TriLocation) module, which utilizes 3D Gaussian Splatting as a continuous scene representation and identifies functional regions under triangular geometric constraints. Finally, the 3D Keypoint Grasp Matrix Transformation Execution (KGT3D+) module decodes these semantic-geometric constraints into physically plausible wrist poses and finger-level commands. Extensive experiments on complex benchmarks demonstrate that BLaDA significantly outperforms existing methods in both affordance grounding precision and the success rate of functional manipulation across diverse categories and tasks. Code will be publicly available at https://github.com/PopeyePxx/BLaDA.

## 二、方法论：框架与技术要点

### 1. 整体框架思想
- 提出了 **BLaDA（Bridging Language to Dexterous Actions in 3DGS fields）**：  
  一个零样本、可解释的模块化系统，用三阶段推理链实现语言到动作的映射：
  1. **语言解析（KLP）** — 将自然语言解析成结构化六元组。  
  2. **三角定位（TriLocation）** — 在 3D Gaussian 场中定位功能点。  
  3. **抓取矩阵变换与执行（KGT3D+）** — 转换为腕部姿态与指级控制命令。
  
### 2. 关键模块与技术细节

#### （1）知识引导语言解析模块（KLP）
- 输入自然语言指令 *L*，输出结构化六元组：
  \[
  S = (g_a, g_r, g_t, g_f, τ, κ)
  \]
  - **g_a**：可供性区域（grasp affordance）  
  - **g_r**：功能手指角色（functional finger）  
  - **g_t**：抓取类型（grasp type）  
  - **g_f**：抓取力度（grasp force）  
  - **τ**：工具拓扑先验（tool topology）  
  - **κ**：任务意图先验（task intent）

- 将通用 LLM（如 ChatGPT4、DeepSeekV3、Gemini2.5）与结构化知识库（F2F）结合，实现**语义到控制约束的可解释解析**。
- 工具拓扑包括四类：轴杆形、侧柄形、旋钮形、平面形；任务意图分为按压、点击、开启、保持。

#### （2）三角功能点定位模块（TriLocation）
- 目标：在 3DGS 场中定位三个关键点 \({p1, p2, p3}\)，定义功能性接触三角结构。
  - **p1：功能接触点（指尖）**
  - **p2：支撑点（小指侧）**
  - **p3：腕部支撑点**

- 技术实现：
  - **Hierarchical Semantic Extraction (HSE)**：基于 YOLO + SAM + CLIP 的层级特征选取、上下文裁剪，避免语义漂移。  
  - 构建多层 3D 高斯场，使用 CLIP 相似度计算定位功能点。
  - 建立 **任务-拓扑约束坐标系**（利用 PCA/RANSAC 提取结构轴、法向量），实现姿态一致性。
  - 使用结构损失约束边长与内角，稳定三角形几何。

#### （3）三维抓取矩阵变换与执行模块（KGT3D+）
- 输入三关键点 → 构建手腕姿态坐标系：
  \[
  R = [\vec{x}, \vec{y}, \vec{z}], \quad T = p_3
  \]
- 由语言解析结果 (g_t, g_f) 查表获得指关节角度 J 与各指力分布 F：
  \[
  (J, F) = F2F(g_t, g_f)
  \]
- 输出物理可执行的姿态与动力指令，实现从语言约束到动作执行的闭环。

## 八、不足与局限

- **语义歧义问题**：3DGS 场对细粒度部件理解仍易出现混淆，如工具“头”与“体”的命名问题。  
- **无触觉闭环**：缺乏触觉传感或位移反馈，可能在执行中因滑动或偏差失效。  
- **场景稀疏限制**：在极其复杂或多物体场景中，高斯场构建易受遮挡与采样影响。  
- **算力可扩展性未验证**：未测试在低算力或轻量平台上的适应性。  
- **任务范围有限**：当前仅覆盖四种基本任务类型（press、click、open、hold），尚未验证长时序操作。
