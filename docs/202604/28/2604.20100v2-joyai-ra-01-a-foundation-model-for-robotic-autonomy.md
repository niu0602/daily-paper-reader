## 摘要
在开放世界环境中，机器人自主性受到数据多样性不足和跨形态泛化性能较差的根本限制。现有的机器人数据集通常在规模和任务覆盖方面有限，而不同机器人形态之间的显著差异阻碍了行为知识的有效迁移。为应对这些挑战，我们提出了 JoyAI-RA，这是一种面向可泛化机器人操作的视觉-语言-动作（VLA）具身基础模型。JoyAI-RA 提出了一种多源多层预训练框架，将网页数据、大规模自我视角的人类操作视频、仿真生成的轨迹以及真实机器人数据相结合。通过在具备显式动作空间统一的异构多源数据上进行训练，JoyAI-RA 有效地弥合了具身差距，尤其是在人类操作与机器人控制之间，从而增强了跨具身的行为学习能力。JoyAI-RA 在仿真和真实世界基准测试中均优于现有最先进的方法，尤其在具有泛化需求的多样化任务上表现突出。

## Abstract
Robotic autonomy in open-world environments is fundamentally limited by insufficient data diversity and poor cross-embodiment generalization. Existing robotic datasets are often limited in scale and task coverage, while relatively large differences across robot embodiments impede effective behavior knowledge transfer. To address these challenges, we propose JoyAI-RA, a vision-language-action (VLA) embodied foundation model tailored for generalizable robotic manipulation. JoyAI-RA presents a multi-source multi-level pretraining framework that integrates web data, large-scale egocentric human manipulation videos, simulation-generated trajectories, and real-robot data. Through training on heterogeneous multi-source data with explicit action-space unification, JoyAI-RA effectively bridges embodiment gaps, particularly between human manipulation and robotic control, thereby enhancing cross-embodiment behavior learning. JoyAI-RA outperforms state-of-the-art methods in both simulation and real-world benchmarks, especially on diverse tasks with generalization demands.

## 2. 方法论与模型设计

### 2.1 核心思想

- 提出**JoyAI-RA 0.1**：一个基于 VLA 架构的**多源多层预训练框架（Multi-Source Multi-Level Pretraining Framework）**；
- 目标：通过融合互补数据源并**显式统一动作空间（Action-Space Unification）**，在不同具身系统之间实现知识迁移；
- 方法核心在于在视觉、语言与动作三模态统一的基础上，建立可迁移、可泛化的操作策略。

### 2.2 关键技术设计

- **四类异构数据源整合**：
  1. 网络多模态数据（提升语义与感知先验）；
  2. 人类第一视角操作视频（EgoLive：自建1万+任务、约1000小时视频）；
  3. 仿真生成轨迹（InternData、GenieSim等）；
  4. 真实机器人操作数据（Open-X-Embodiment、AgiBot-World、自采JDAgibot）。

- **统一动作空间（Unified Action Space）**：
  - 将不同机器人动作映射至**相机坐标系的6-DoF末端执行器空间**；
  - 使用固定维度动作向量+掩码机制，兼容多机械臂（单臂/双臂/手指）；
  - 提升了跨形态数据的一致性与物理语义可迁移性。

- **模型结构**（见 Fig. 3）：
  - **VLM子模块（4B参数）**：处理视觉-语言理解；
  - **Perceiver-based Action Expert（约6亿参数）**：执行连续动作生成；
  - 两者通过语义特征连接，先生成高层语义 token 再生成连续动作序列。

- **训练流程（分三阶段）**：
  1. **VLM共预训练**：学习视觉理解、空间推理与文本生成；
  2. **VLA共预训练**：整合人类视频、仿真与机器人数据完成动作预测；
  3. **后训练阶段（Post-Training）**：面向具体机器人（如AgiBot G1）进行轻微微调。

- **训练目标函数**（简述）：
  - 阶段一：最小化自回归负对数似然；
  - 阶段二与三：加入**流匹配（Flow-Matching）目标**，在噪声空间预测动作速度场，提高动作时序一致性。
